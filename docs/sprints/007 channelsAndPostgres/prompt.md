Claude Code has channels (see https://code.claude.com/docs/en/channels). I want it so that questions that are asked in a Claude Code session are pushed to the decision broker postgres via channels. A specific Claude Code session should create a new board if it does not have one. The question creates a new decision item. If there are many questions, each question is a seperate decision item. When a user submits an answer, the answer is returned to the Claude Code session. The idea is that decision broker has a better user interface/is more multiplayer than a simple terminal with Claude Code open. The Claude Code session should not continue with whatever it is working on until all questions are answered.
